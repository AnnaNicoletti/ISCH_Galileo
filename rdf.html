A semantic representation of Galileo-related cultural heritage resources

Metodology

This RDF dataset describes a curated selection of ten cultural heritage items related to Galileo Galilei, such as books, people, audiovisual materials, historical events, and cultural artifacts. All metadata were structured and semantically enriched using Linked Open Data principles and ontologies such as schema.org, Dublin Coreand CIDOC-CRM.

Each item was initially described in a CSV file, containing structured metadata mapped to controlled vocabularies and authority records (e.g., VIAF, Wikidata). These CSV files were then transformed into RDF using Python scripts and the rdflib library.

From CSV to RDF

Following the CSV-to-RDF pipeline described above, we generated one Turtle file for each of the ten cultural heritage items.These files represent the first RDF output of our workflow: each CSV source was mapped to RDF triples, enriched with controlled vocabularies and authority URIs. Next, we moved on to the special case of the full-text TEI edition, transforming it into RDF as well.

From TEI to RDF

The full text of the Dialogo sopra i due massimi sistemi del mondo was encoded in TEI/XML following the TEI P5 guidelines. We then used a Python script with rdflib to parse the XML, extract entities (persons, schools of thought, locations) and structural divisions, and serialize everything as RDF/Turtle. The TEI file was analyzed to identify and extract key semantic elements. Named entities such as persons, places, and schools of thought were mapped using vocabularies such as schema.org and Dublin Core. Structural divisions like chapters and speaker turns were also represented as RDF classes and linked to their content.
